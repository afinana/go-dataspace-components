# ADR-0001: Record Architecture Decisions

## Status

Accepted

## Context

We need to record architectural decisions made for the Sovereign Dataspace Connector project so that future developers (and AI agents) understand the design rationale, constraints, and operational guidelines.

## Decision

We will use Architecture Decision Records (ADRs) to document significant design decisions. They will be stored in the `/docs/adr/` folder using standard Markdown format.

## Consequences

*   Decisions are version-controlled alongside source code.
*   Context and rationale for design decisions remain accessible over time.
*   New developers/agents can quickly ramp up on design guidelines.
